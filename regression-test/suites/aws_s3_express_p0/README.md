# S3 Express One Zone import regression test

This suite validates importing existing objects from an Amazon S3 Express One Zone directory bucket. It covers S3 TVF queries, `INSERT INTO ... SELECT ... FROM S3(...)`, Broker Load, canonical Object URLs, `s3://` URIs, CSV, Parquet, ORC, multi-path and glob discovery, and `ListObjectsV2` pagination over 1001 objects.

The suite does not create a directory bucket or write through Doris. Run it manually against a real AWS environment. It is skipped unless `enableS3ExpressOneZoneTest` is set to `true`.

## Prerequisites

- Create the directory bucket separately and choose a new, empty test prefix.
- The IAM credentials used by Doris and the AWS CLI must allow `s3express:CreateSession` for that bucket.
- The FE and every BE must be able to reach the bucket's zonal endpoint.
- Install an AWS CLI v2 version that supports S3 directory buckets.
- If the bucket uses SSE-KMS, grant the corresponding KMS permissions to the fixture uploader and Doris IAM principal.

The test configuration selects an existing Doris IAM credential provider without putting AK/SK or tokens in SQL or regression logs. These are not the short-term S3 Express session credentials returned by `CreateSession`; the AWS SDK creates and refreshes those internally. Use a dedicated least-privilege IAM principal.

## Prepare the existing objects

The preparation script only uploads deterministic fixtures into the configured prefix of an existing directory bucket. It refuses a non-empty prefix and does not create or delete a bucket or clean up objects. If preparation is interrupted, choose another new prefix or clean up the partial fixture manually before retrying.

```bash
export S3_EXPRESS_BUCKET='example--usw2-az1--x-s3'
export S3_EXPRESS_REGION='us-west-2'
export S3_EXPRESS_PREFIX='doris-regression/s3-express-one-zone'

./regression-test/suites/aws_s3_express_p0/prepare_s3_express_data.sh
```

The AWS CLI uses its normal credential provider chain. The script uploads three matching CSV objects, one non-matching CSV sibling, one Parquet object, one ORC object, and 1001 pagination objects.

## Configure the regression runner

Copy `regression-test/conf/regression-conf.groovy` to a private file and add:

```groovy
enableS3ExpressOneZoneTest = "true"
s3ExpressBucket = System.getenv("S3_EXPRESS_BUCKET")
s3ExpressRegion = System.getenv("S3_EXPRESS_REGION")
s3ExpressPrefix = System.getenv("S3_EXPRESS_PREFIX")
s3ExpressCredentialsProviderType = "INSTANCE_PROFILE"
```

With `INSTANCE_PROFILE`, the FE and every BE machine must have an attached IAM role. Other non-secret provider modes supported by Doris can be used, but their prerequisites must exist in every FE and BE process. For example, `ENV` requires the AWS credential environment variables on the FE and every BE, not only in the shell that starts the regression runner.

## Run on AWS

The expected output must be generated by the first real AWS run; do not handwrite it. After reviewing the generated output, run the suite again without `-genOut`:

```bash
./run-regression-test.sh --run --conf /absolute/path/to/aws-s3-express-conf.groovy \
    -d aws_s3_express_p0 -s test_s3_express_import -genOut

./run-regression-test.sh --run --conf /absolute/path/to/aws-s3-express-conf.groovy \
    -d aws_s3_express_p0 -s test_s3_express_import
```

Commit `regression-test/data/aws_s3_express_p0/test_s3_express_import.out` only after both commands pass on the real AWS directory bucket.
