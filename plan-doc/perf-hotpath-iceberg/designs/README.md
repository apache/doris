# per-task 设计 + 小结

每个 `PERF-NN` 任务在**实现时**（不预建）产出两份文档：

- `FIX-PERF-NN-<slug>-design.md` —— Problem / Root Cause / Design / Implementation Plan / Risk / Test Plan（含收益度量方案）
- `FIX-PERF-NN-<slug>-summary.md` —— Problem / Root Cause / Fix / Tests / Result

命名对齐仓库既有惯例（`plan-doc/designs/FIX-*-design.md`、`plan-doc/tasks/designs/FIX-*-design.md`）。
`<slug>` 用短横线短语，例：`FIX-PERF-01-table-memo-design.md`。

立项流程见 [`../README.md`](../README.md) §单项立项流程。
